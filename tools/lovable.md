# 💜 Lovable.dev – Fördjupning
*Status: Aktivt utvärderad (Mars 2026)*

## 🎯 Varför Lovable?
Lovable är inte bara en "chatt-bott" som skriver kod; det är en orchestrator som bygger en fullskalig webbapplikation med en modern tech-stack. Det är det snabbaste sättet att gå från en idé till en produkt som användare kan logga in i.

## 🛠 Teknisk Stack
När du bygger med Lovable skapas följande under huven:
* **Frontend:** React med Vite (snabbt och modernt).
* **Styling:** Tailwind CSS (skalbart och lätt att justera).
* **UI-komponenter:** Shadcn/ui (professionell standard).
* **Backend/Databas:** Supabase (PostgreSQL, Auth, Storage).

## 💡 Strategi för LL (Op. Rainbow Road)
För att integrera Lovable med ditt bygge inom kodning och IT (**LL**), bör du följa denna workflow:

1. **Prompting:** Beskriv logiken för din app i Lovable-chatten.
2. **Databas-koppling:** Anslut ditt Supabase-projekt direkt i Lovable för att spara riktig data.
3. **GitHub-sync:** Koppla repot till GitHub (detta repo!) så att varje ändring i Lovable skapar en "commit".
4. **Anytype-logg:** Dokumentera arkitekturen du skapar här i din Anytype-library under koden **LL**.

## 🚀 Expert-tips för navigering
* **Edit-mode:** Använd "Edit mode" för att peka på specifika delar av skärmen du vill ändra.
* **Custom Instructions:** Du kan ge Lovable instruktioner om att alltid använda en viss designmanual eller kodstandard.
* **Rollback:** Om en ändring förstör något, använd historik-funktionen för att backa till en stabil version.
